# AskBridge 기획 문서

> AI와 협업하여 작성한 서비스 기획 문서입니다.

---

## 1. 서비스 개요

**AskBridge**는 교강사가 수업 자료를 업로드하면 AI가 학습 목표, 예상 질문, 단답형 퀴즈를 자동 생성하고 학생에게 즉시 공유할 수 있는 교육용 AI 솔루션입니다.

```
교사: 수업 자료 업로드 → AI 분석 → 질문/퀴즈 선택 → 공유 링크 전송
학생: 링크 접속 → 학습 목표 확인 → 예상 질문 열람 → 퀴즈 풀기
```

---

## 2. 타겟 사용자 및 문제점

### 교강사

교강사는 수업 중 두 가지 문제를 자주 마주한다.

**문제 1: 학생의 침묵**
학생들은 이해가 안 되는 부분이 있어도 질문하기를 꺼린다. 이로 인해 강사는 어떤 부분이 실제로 어렵게 느껴지는지 파악하기 어렵고, 중요한 개념을 그냥 지나치게 된다. AskBridge는 AI가 예상 질문을 미리 생성하여 강사가 수업 중 선제적으로 짚어줄 수 있도록 돕는다.

**문제 2: 강의 자료 완성도 검증**
강사는 자신이 만든 자료가 학습 목표에 맞게 잘 구성되었는지, 어떤 개념이 모호하거나 오해를 일으킬 수 있는지 스스로 판단하기 어렵다. AI가 생성한 예상 질문은 자료의 취약한 부분을 객관적으로 드러내어 강사가 자료를 개선하는 데 활용할 수 있다.

### 학생

수업 중 이해가 안 되는 부분이 있어도 **"나만 모르는 것 같다"**, **"질문하면 수업 흐름이 끊길 것 같다"** 는 심리적 부담으로 질문을 참는 경우가 많다. 결국 모르는 채로 수업이 끝나고, 혼자 해결하거나 그냥 넘어가게 된다.

또한 수업 전 학습 목표를 미리 파악하거나 수업 후 퀴즈로 복습하고 싶지만 별도 플랫폼 가입·앱 설치 없이 접근할 수단이 없다.

## 3. 교사 / 학생 역할 분리

| 기능 | 교사 | 학생 |
|------|------|------|
| 학습 목표 | 자료 완성도 검토 | 수업 전 예습 |
| 예상 질문 | 강의 보완 포인트 파악, 수업 중 선제적 설명 | — |
| 퀴즈 | 형성평가 문항 제작, 인쇄 배포 | 수업 마무리 리마인드, 다음 시간 복습 도입 |

## 4. 핵심 기능 설계 의도

### 다양한 파일 포맷 지원
PDF, PPTX, DOCX, TXT, MD, CSV, IPYNB를 지원하여 교육 현장에서 실제로 사용하는 모든 자료 형식을 커버한다. 특히 IPYNB는 데이터 사이언스·프로그래밍 수업을 겨냥한 차별점이다.

### 대상 수준 설정 (입문 / 기초 / 중급 / 심화)
동일한 자료라도 수강생 수준에 따라 필요한 질문과 설명 깊이가 다르다. 수준별 시스템 프롬프트를 동적으로 주입하여 맞춤형 결과를 생성한다.

### 교사 큐레이션 (체크박스 선택)
AI가 생성한 결과를 그대로 사용하지 않고 교사가 직접 선별할 수 있도록 설계했다. 교육 콘텐츠의 최종 판단은 교사에게 있다는 철학을 반영한다.

### 서버리스 공유 링크 (LZString URL 압축)
별도 DB나 서버 없이 선택한 콘텐츠를 URL로 압축하여 학생에게 링크 하나로 전달한다. 교육 현장에서 즉시 사용 가능하도록 도입 장벽을 최소화하는 데 초점을 맞췄다.

### 인쇄 지원 (정답 포함 / 제외)
오프라인 수업 환경을 고려하여 정답이 포함된 버전과 제외된 버전을 선택해 인쇄할 수 있다. 섹션별 페이지 분리로 배포 자료로 바로 활용 가능하다.

## 5. 기대 효과

**교강사**
- 예상 질문을 미리 파악해 학생이 질문하지 않아도 능동적으로 짚어줄 수 있음 → 학생의 심리적 질문 장벽 우회
- AI 예상 질문으로 강의 자료의 모호한 부분을 객관적으로 파악 → 자료 품질 향상
- 예상 질문·퀴즈 제작 시간 수십 분 → 수 분으로 단축

**학생**
- 수업 전: 학습 목표로 무엇을 배울지 예습하여 능동적 학습 태도 형성
- 수업 중/후: 퀴즈로 이해도 즉시 자기 점검 (수업 마무리 리마인드)
- 다음 수업 시작 시: 이전 시간 퀴즈로 복습 도입부로 활용

**교육 현장**
- 별도 플랫폼 가입·설치 없이 링크 하나로 즉시 배포 → 현장 도입 장벽 최소화

## 6. AI 활용 전략

### 사용 모델
- **개발 보조**: Claude Code (claude-sonnet-4-6)
- **서비스 내 AI**: Anthropic API — claude-sonnet-4-6

### 프롬프트 설계
- 단일 API 호출로 학습 목표 + 예상 질문 + 퀴즈를 구조화된 JSON으로 동시 생성
- 각 질문에 출제 이유, 모범 답변, 난이도, 출처 섹션을 포함하여 교사가 결과를 신뢰하고 활용할 수 있도록 설계
- 대상 수준(입문/기초/중급/심화)에 따라 시스템 프롬프트 컨텍스트를 동적 주입

### 토큰 최적화
- **프롬프트 캐싱**: `cache_control: { type: 'ephemeral' }` 적용으로 동일 수준 반복 호출 시 입력 토큰 비용 절감
- **입력 길이 제한**: 20,000자 초과 시 프론트엔드에서 차단하여 불필요한 API 호출 방지
- **스트리밍**: 응답 완료 전에 생성된 항목부터 순차 표시하여 사용자 체감 속도 향상

### Claude Code 협업 방식
`CLAUDE.md` 프로젝트 지침 파일에 작업 규칙을 명문화하여 세션이 바뀌어도 일관된 협업 맥락을 유지했다.
- 질문 시 바로 코딩하지 않고 편법/정석 옵션 제시 후 명시적 컨펌을 받는 방식으로 불필요한 대규모 변경 방지
- 시스템 프롬프트 수정 시 before/after diff 확인 후 명시적 컨펌 필수
- 모든 코드 수정 후 빌드 검증 및 서버 재시작을 의무화하여 배포 품질 유지
